Complete guide to building a Solana flash loan arbitrage bot — price monitoring, profit detection, on-chain Anchor program, transaction construction, Jito MEV bundles, and submission optimization.

End-to-end system design

How the bot works

A Solana arb bot has four distinct layers: an off-chain monitor that watches AMM pool accounts in real time, a profit calculator that filters opportunities, a transaction builder that constructs the atomic multi-instruction tx, and a submitter that wins block inclusion via priority fees or Jito bundles.

System pipeline

1. Monitor
WebSocket subscription to pool accounts. Decode reserves on every account change event.
2. Detect
Compute spot prices. Calculate net profit after all fees. Filter if below threshold.
3. Build tx
Construct: compute budget → flash borrow → swap A → swap B → flash repay.
4. Submit
Send via Jito bundle with tip. Broadcast to multiple RPCs for coverage.
5. Capture
Profit deposited to wallet. Monitor for confirmation. Adjust tip for next opportunity.

Real-time pool state subscription

Watching AMM reserves as they change

Every time a swap happens in a pool, the pool's on-chain account data changes. You subscribe to these account changes via WebSocket and decode the raw bytes into reserve amounts. Speed here is everything — you need to react before anyone else.

monitor.ts — subscribe to Orca and Raydium pool accounts simultaneously
import { Connection, PublicKey, AccountInfo, Context } from '@solana/web3.js'; import { WhirlpoolData } from '@orca-so/whirlpools-sdk'; import { LiquidityStateV4 } from '@raydium-io/raydium-sdk-v2'; import * as BN from 'bn.js'; // ── Pool addresses for SOL/USDC pair on both DEXes ── const ORCA_SOL_USDC_WHIRLPOOL = new PublicKey('HJPjoWUrhoZzkNfRpHuieeFk9WcZWjwy6PBjZ81ngndJ'); const RAYDIUM_SOL_USDC_POOL = new PublicKey('58oQChx4yWmvKdwLLZzBi4ChoCc2fqCUWagsenfME4NG'); const USDC_DECIMALS = 6; const SOL_DECIMALS = 9; interface PoolPrice { dex: string; pool: PublicKey; priceUsdc: number; // SOL price in USDC sqrtPrice: BN; timestamp: number; } class PriceMonitor { private prices = new Map<string, PoolPrice>(); private onOpportunity: (opp: ArbOpportunity) => void; constructor( private conn: Connection, onOpp: (o: ArbOpportunity) => void ) { this.onOpportunity = onOpp; } async start() { // Subscribe to Orca Whirlpool account changes this.conn.onAccountChange( ORCA_SOL_USDC_WHIRLPOOL, (info, ctx) => this.handleOrca(info, ctx), 'processed' // fastest commitment — not yet confirmed ); // Subscribe to Raydium AMM pool account changes this.conn.onAccountChange( RAYDIUM_SOL_USDC_POOL, (info, ctx) => this.handleRaydium(info, ctx), 'processed' ); console.log('Subscribed to pools. Monitoring for opportunities...'); } private handleOrca(info: AccountInfo<Buffer>, ctx: Context) { const pool = WhirlpoolData.decode(info.data); // Orca SDK decodes layout // Orca uses Q64.64 sqrt price — convert to human-readable const sqrtPrice = pool.sqrtPrice; const rawPrice = sqrtPrice.toNumber() ** 2 / (2 ** 64); const decimalAdj = 10 ** (USDC_DECIMALS - SOL_DECIMALS); const priceUsdc = rawPrice / decimalAdj; this.prices.set('orca', { dex: 'orca', pool: ORCA_SOL_USDC_WHIRLPOOL, priceUsdc, sqrtPrice, timestamp: Date.now() }); this.checkOpportunity(); } private handleRaydium(info: AccountInfo<Buffer>, ctx: Context) { const pool = LiquidityStateV4.decode(info.data); // Raydium SDK // Raydium constant-product: price = baseVault / quoteVault (adjusted for decimals) const baseAmt = pool.baseVault.toNumber() / 10 ** SOL_DECIMALS; const quoteAmt = pool.quoteVault.toNumber() / 10 ** USDC_DECIMALS; const priceUsdc = quoteAmt / baseAmt; // USDC per SOL this.prices.set('raydium', { dex: 'raydium', pool: RAYDIUM_SOL_USDC_POOL, priceUsdc, sqrtPrice: new BN(0), timestamp: Date.now() }); this.checkOpportunity(); } private checkOpportunity() { const orca = this.prices.get('orca'); const raydium = this.prices.get('raydium'); if (!orca || !raydium) return; // need both prices // Stale data guard — discard prices older than 2 slots (800ms) if (Date.now() - Math.min(orca.timestamp, raydium.timestamp) > 800) return; const priceDiff = Math.abs(orca.priceUsdc - raydium.priceUsdc); const midPrice = (orca.priceUsdc + raydium.priceUsdc) / 2; const pctDiff = priceDiff / midPrice; console.log(`Orca: $${orca.priceUsdc.toFixed(4)} Raydium: $${raydium.priceUsdc.toFixed(4)} Δ: ${(pctDiff*100).toFixed(4)}%`); if (pctDiff > 0.003) { // 0.3% minimum gap (covers typical fees) this.onOpportunity({ orca, raydium, pctDiff, midPrice }); } } }

The math that decides whether to fire

Calculating net profit before submitting

Firing on every price discrepancy loses money. You must account for every cost — swap fees on both sides, flash loan fee, Jito tip, priority fee, and slippage — before deciding to submit. The detection logic is the heart of a profitable bot.

profit-calculator.ts — exact profit math with all cost deductions
const FEE = { ORCA_WHIRLPOOL_BPS: 4, // 0.04% (tight pool tier) RAYDIUM_AMM_BPS: 25, // 0.25% FLASH_LOAN_BPS: 0, // 0% on Marginfi flash loans JITO_TIP_SOL: 0.003,// competitive Jito tip (adjust dynamically) PRIORITY_FEE_LAMPORTS:200_000, // 0.0002 SOL compute unit fee }; interface ArbOpportunity { buyDex: 'orca' | 'raydium'; // buy SOL here (lower price) sellDex: 'orca' | 'raydium'; // sell SOL here (higher price) buyPrice: number; // USDC/SOL sellPrice: number; // USDC/SOL optimalAmtSol: number; netProfitUsdc: number; } function calcOptimalArb( buyPool: PoolState, sellPool: PoolState, solPriceUsdc: number ): ArbOpportunity | null { // ── Optimal trade size using constant-product formula ── // For two pools with reserves (rIn_A, rOut_A) and (rIn_B, rOut_B): // Optimal input = sqrt(rIn_A * rIn_B * rOut_A / rOut_B) - rIn_A // (Maximizes profit accounting for price impact on both pools) const rSolBuy = buyPool.solReserve; const rUsdcBuy = buyPool.usdcReserve; const rSolSell = sellPool.solReserve; const rUsdcSell= sellPool.usdcReserve; const optimalSol = Math.sqrt( (rSolBuy * rUsdcBuy * rSolSell) / rUsdcSell ) - rSolBuy; if (optimalSol <= 0 || optimalSol > 500) return null; // cap at 500 SOL // ── Apply swap fee on buy leg (pay USDC, receive SOL) ── const usdcIn = getAmountIn(buyPool, optimalSol, 'USDC→SOL', FEE.ORCA_WHIRLPOOL_BPS); // ── Apply swap fee on sell leg (pay SOL, receive USDC) ── const usdcOut= getAmountOut(sellPool, optimalSol, 'SOL→USDC', FEE.RAYDIUM_AMM_BPS); // ── Gross profit ── const grossProfit = usdcOut - usdcIn; // ── Subtract fixed costs (convert all to USDC) ── const jitoTipUsdc = FEE.JITO_TIP_SOL * solPriceUsdc; const priorityFeeUsdc = (FEE.PRIORITY_FEE_LAMPORTS / 1e9) * solPriceUsdc; const flashLoanFeeUsdc= usdcIn * (FEE.FLASH_LOAN_BPS / 10000); const totalCost = jitoTipUsdc + priorityFeeUsdc + flashLoanFeeUsdc; const netProfit = grossProfit - totalCost; console.log(` Trade size: ${optimalSol.toFixed(4)} SOL USDC in: $${usdcIn.toFixed(4)} USDC out: $${usdcOut.toFixed(4)} Gross profit: $${grossProfit.toFixed(4)} Jito tip: $${jitoTipUsdc.toFixed(4)} Priority fee: $${priorityFeeUsdc.toFixed(4)} NET PROFIT: $${netProfit.toFixed(4)} ${netProfit > 0 ? '✅ FIRE' : '❌ SKIP'} `); if (netProfit < 0.50) return null; // minimum $0.50 profit threshold return { buyDex: 'orca', sellDex: 'raydium', buyPrice: buyPool.price, sellPrice: sellPool.price, optimalAmtSol: optimalSol, netProfitUsdc: netProfit, }; } // x*y=k constant-product output formula with fee function getAmountOut( pool: PoolState, amtIn: number, direction: string, feeBps: number ): number { const [rIn, rOut] = direction === 'SOL→USDC' ? [pool.solReserve, pool.usdcReserve] : [pool.usdcReserve, pool.solReserve]; const amtInWithFee = amtIn * (10000 - feeBps); return (rOut * amtInWithFee) / (rIn * 10000 + amtInWithFee); }
Slippage simulation: in practice you should also simulate the transaction with simulateTransaction() before sending. This catches cases where the pool state changed between detection and submission, preventing you from losing money on a bad trade.

Rust/Anchor — the atomic arbitrage program

The on-chain logic: borrow → swap → swap → repay

Your Anchor program executes the arbitrage atomically. If any step fails — wrong price, insufficient funds, slippage exceeded — the entire transaction reverts and you pay nothing (except the priority fee). This is the safety guarantee that makes flash loan arb low-risk.

Constructing the optimal transaction

Versioned transactions, compute budget, and lookup tables

A flash loan arb transaction touches many accounts — flash loan program, two DEX programs, multiple vaults, tick arrays, oracles. This pushes past Solana's 32-account limit for legacy transactions. Versioned transactions with Address Lookup Tables solve this and also reduce transaction size, lowering fees.

tx-builder.ts — construct the full arbitrage transaction

Winning block inclusion — the final mile

Jito bundles, priority fees, and latency

Detection and construction mean nothing if another bot lands their transaction first. Jito's block engine lets you submit bundles with guaranteed ordering. Tip auctions are ruthless — your tip must exceed competitors' or your bundle is skipped. Here's how to win.

submitter.ts — Jito bundle submission with dynamic tip bidding
import { searcherClient, Bundle } from 'jito-ts/dist/sdk/block-engine/searcher'; import { isError } from 'jito-ts/dist/sdk/block-engine/utils'; const JITO_BLOCK_ENGINE_URL = 'mainnet.block-engine.jito.wtf:443'; class ArbSubmitter { private searcher: any; private tipHistory: number[] = []; // track winning tips to calibrate async connect(authKeypair: Keypair) { this.searcher = await searcherClient( JITO_BLOCK_ENGINE_URL, authKeypair // searcher identity for rate limits ); // Subscribe to bundle outcomes — learn what tips are winning this.searcher.onBundleResult( (result: any) => { if (result.accepted) console.log('✅ Bundle accepted, slot:', result.accepted.slot); if (result.rejected) console.log('❌ Bundle rejected:', result.rejected.stateAuctionBidRejected?.msg); if (result.droppedBundle) console.log('⚠️ Bundle dropped (slot missed)'); }, (err: Error) => console.error('Bundle stream error:', err) ); } async submit( arbTx: VersionedTransaction, profitUsdc: number, solPriceUsdc:number ): Promise<boolean> { // ── Dynamic tip: bid 60% of profit, minimum 0.001 SOL ── // If profit is $5 and SOL = $150, tip up to ~0.02 SOL // Adjust based on competition — track rejected bundles const profitSol = profitUsdc / solPriceUsdc; const targetTip = Math.max(profitSol * 0.6, 0.001); const tipLamports = Math.floor(targetTip * LAMPORTS_PER_SOL); const bundle = new Bundle([arbTx], 5 /* max txs in bundle */); try { const resp = await this.searcher.sendBundle(bundle); if (isError(resp)) { console.error('Bundle error:', resp); return false; } console.log(`Bundle sent: ${resp} tip: ${(tipLamports/LAMPORTS_PER_SOL).toFixed(5)} SOL`); return true; } catch (e) { console.error('Send failed:', e); return false; } } } // ── Parallel broadcast: send to Jito AND regular RPCs simultaneously ── // If Jito leader isn't the next slot, regular broadcast wins. // Belt-and-suspenders approach used by most production bots. async function broadcastParallel( tx: VersionedTransaction, conn: Connection, submitter:ArbSubmitter, profitUsdc:number, solPrice: number ) { const rawTx = tx.serialize(); await Promise.allSettled([ // Path A: Jito bundle (guaranteed ordering, MEV protection) submitter.submit(tx, profitUsdc, solPrice), // Path B: Primary RPC (low-latency connection) conn.sendRawTransaction(rawTx, { skipPreflight: true, // don't simulate — already did that maxRetries: 0, // no auto-retry — stale by the time it's retried }), // Path C: Secondary RPC for coverage backupConn.sendRawTransaction(rawTx, { skipPreflight: true, maxRetries: 0 }), ]); }

Latency optimization checklist

Infrastructure

Bare-metal server in the same datacenter as major validators (Frankfurt, Amsterdam, Tokyo, Ashburn). Each 10ms of round-trip time costs you roughly 2.5% of the slot window.

RPC choice

Use Triton RPC, Helius, or QuickNode for dedicated nodes. Never use the public mainnet-beta.solana.com endpoint — it's rate-limited and slow.

Pre-build transactions

Pre-sign partial transactions (everything except blockhash) at startup. On opportunity, only update the blockhash and slippage params — saves 10–30ms of construction time.

Write detection in Rust

TypeScript adds 5–20ms overhead vs native Rust for pool decoding and profit math. Production bots use Rust for the hot path and TypeScript only for config and monitoring.

Tip auction strategy

Start with 50–60% of expected profit as Jito tip. Track rejection reasons. Increase tip if frequently rejected on high-profit opportunities. Decrease if winning easily.

Always simulate first

Call simulateTransaction() with replaceRecentBlockhash: true before every live submission. A failed on-chain tx still costs priority fees + Jito tip.

The most profitable strategy today is not raw cross-DEX arbitrage (extremely competitive) but rather finding and exploiting price inefficiencies in newly-launched token pools, low-liquidity long-tail pairs, or cross-chain price divergences (Solana vs EVM via Wormhole prices) — areas where fewer bots are looking.
