Core pillars

Decentralized

No single entity controls the network. Data is replicated across thousands of independent nodes globally.

Permissionless

Anyone can participate without approval from a central authority — no sign-ups, no gatekeepers.

Trustless

Smart contracts enforce rules via cryptographic math — no need to trust a company or institution.

Self-sovereign

You own your identity and digital assets through cryptographic keys — not a company's password policy.

The foundational layer

How blockchain works

A distributed ledger — an ordered chain of cryptographically linked blocks, simultaneously replicated across thousands of independent nodes worldwide.

Changing block #2 alters its hash → invalidates block #3's "Prev" pointer → cascades through the entire chain. Tampering is computationally infeasible at scale.

Four defining properties

Immutability

Written records cannot be changed without rewriting all subsequent blocks — requiring more computation than the entire honest network has produced.

Decentralization

Thousands of nodes each hold a complete ledger copy. No single server to hack, censor, or shut down — the network is the server.

Transparency

All transactions are permanently public. Block explorers like Etherscan let anyone inspect every transaction and smart contract interaction ever made.

Pseudonymity

Transactions reference wallet addresses, not real names. Addresses are fully traceable on-chain but not directly linked to identity by default.

Consensus mechanisms

Proof of Work

PoW

Miners race to solve SHA-256 puzzles. First to solve adds the block and earns the reward. Extremely secure but energy-intensive.

Bitcoin · Litecoin · Monero

Proof of Stake

PoS

Validators lock tokens as collateral. Selected to propose blocks proportional to stake. ~99.95% less energy than PoW.

Ethereum · Cardano · Solana

Delegated PoS

DPoS

Token holders elect a small delegate set who validate on their behalf. More efficient but fewer validators means less decentralization.

EOS · TRON · BNB Chain

The logic layer

Smart contracts

Programs deployed to a blockchain that execute automatically when predefined conditions are met — immutable once live, run identically by every node on the network with no intermediary.

Lifecycle

1. Write
Code in Solidity (EVM) or Rust. Define state variables, functions, events.
2. Compile
Source → EVM bytecode + ABI. The ABI tells clients how to call the contract.
3. Deploy
Send a tx with no "to" address. Contract is permanently assigned an on-chain address.
4. Call
Users send transactions to the contract. Every node runs the matching function identically.
5. Execute
Conditions checked. State changes and transfers happen atomically. Recorded forever.

A minimal token contract (Solidity)

// SPDX-License-Identifier: MIT pragma solidity ^0.8.20; contract MyToken { string public name = "MyToken"; uint256 public totalSupply; mapping(address => uint256) public balanceOf; // on-chain storage constructor(uint256 supply) { totalSupply = supply; balanceOf[msg.sender] = supply; // mint to deployer } function transfer(address to, uint256 amount) external { require(balanceOf[msg.sender] >= amount, "Insufficient balance"); balanceOf[msg.sender] -= amount; balanceOf[to] += amount; emit Transfer(msg.sender, to, amount); } event Transfer(address indexed from, address indexed to, uint256 value); }

Gas — the cost of computation

What is gas?

A unit measuring EVM computational work. Every opcode (store, add, call) costs a fixed amount of gas. Prevents infinite loops and spam attacks.

Gas price (gwei)

Since EIP-1559: base fee (burned permanently) + optional priority tip to validator. Higher tip = faster inclusion. Base fee adjusts per block automatically.

Gas limit

Max gas you authorize. If execution exceeds it, the tx reverts — but gas already consumed is not refunded. Set ~20% above the estimate.

Token standards (EVM)

ERC-20

Fungible tokens

Standard for interchangeable tokens. 1 USDC always = 1 USDC. Powers currencies, governance tokens, and all DeFi assets.

ERC-721

Non-fungible tokens

Each token has a unique ID and is non-interchangeable. Standard for art, collectibles, game items, ENS names, real-world asset deeds.

ERC-1155

Multi-token standard

Both fungible and non-fungible tokens in a single contract. Ideal for games (currency + unique items in one deploy). Batch transfer support.

ERC-4626

Tokenized vaults

Standard interface for yield-bearing vaults. Makes DeFi protocols composable — deposit into any vault using the same interface. Used across lending protocols.

Key DeFi risks

Smart contract exploits

Code bugs can drain entire protocols. Billions lost to reentrancy, logic errors, and access control flaws. Always check audit reports before depositing.

Flash loan attacks

Borrow millions uncollateralized in one atomic transaction, manipulate an on-chain price oracle, exploit a protocol, repay — all in a single tx with no upfront capital.

Regulatory risk

Regulators may classify DeFi protocols as unregistered securities exchanges, creating significant legal uncertainty for users and developers globally.

Non-fungible tokens

NFTs — provable digital ownership

A cryptographic certificate of authenticity and ownership for a unique asset — recorded on-chain, globally transferable without any intermediary, and verifiable by anyone.

Fungible vs non-fungible

Fungible — ERC-20

Interchangeable units

Every unit is identical. 1 ETH = 1 ETH. Like dollar bills — it doesn't matter which specific unit you hold, only how many.

Token A=Token B
Non-fungible — ERC-721

Unique identifiers

Each token has a unique ID. Like a deed to a specific house — #001 is definitively not #002, even on the same contract.

#001≠#002

What's actually stored in an NFT?

On-chain (permanent)

Token ID · Owner address · Contract address · Full transfer history · Royalty basis points

Metadata URI (often off-chain)

Name · Description · Trait attributes · Image URL pointing to IPFS or a centralized server

Risk

If the image lives on a centralized server, it can disappear. IPFS-pinned or fully on-chain SVG NFTs are far more durable.

Use cases beyond digital art

Identity & credentials

Soulbound tokens (non-transferable) for diplomas, licenses, reputation scores, and on-chain attestations.

Gaming assets

Truly own in-game items — trade, sell, or use across games. Real economic value that persists beyond a single title's servers.

Tickets & memberships

Unforgeable event tickets. Membership passes with on-chain access gates and built-in royalties on every resale.

Real-world assets (RWA)

Tokenize real estate, fine art, commodities. Enables fractional ownership of expensive assets on-chain via ERC-20 splits of ERC-721 deeds.

Music & royalties

Artists sell royalty rights directly to fans. Smart contracts auto-distribute revenue to multiple rights-holders on every stream or sale.

ENS domains

Turn 0x742d…f44e into alice.eth. Human-readable addresses, decentralized website hosting, and universal Web3 username.

Governance flow

1

Proposal submitted

Any token holder above a threshold (e.g. 1% of supply) submits a governance proposal on-chain or via Snapshot.

2

Discussion period

Community debates in forums (Discourse), Discord, and governance platforms like Tally or Commonwealth. Typically 3–7 days.

3

Voting window

Token holders cast votes. Weight proportional to tokens held (or quadratic voting to limit whale dominance). Common window: 3–7 days. Quorum required.

4

Timelock (safety delay)

Passed proposals enter a timelock (e.g. 48h) allowing users to exit before contentious changes take effect.

5

On-chain execution

Smart contract executes automatically — treasury transfers, parameter changes, protocol upgrades. No human signature required.

Notable DAOs

MakerDAO

Governs the DAI stablecoin. MKR holders vote on collateral types, stability fees, and risk parameters. One of the oldest and largest DAOs.

Uniswap DAO

Controls the Uniswap protocol treasury (over $1B). UNI holders vote on fee switches, grants, and cross-chain deployments.

Nouns DAO

One Noun NFT auctioned daily, forever. Every Noun = one vote on the treasury. A fully on-chain governance experiment with no roadmap.

Cryptography & custody

Wallets, keys & L2 scaling

A wallet doesn't "store" crypto — it stores private keys that prove ownership. Assets live on-chain; your keys are the unforgeable signature. Lose them, lose everything, permanently.

Public-key cryptography

Private key

A 256-bit secret. Signs transactions to prove authorization. Never share with anyone. If lost or stolen, all funds are gone — no recovery exists.

4c0883a69102937d6231471b5dba2b…

Public key

Derived from private key via elliptic curve math (secp256k1). Mathematically irreversible. Hashed to produce your wallet address. Safe to share.

04 a1b2c3d4e5f6… (65 bytes uncompressed)

Wallet address

Keccak-256 hash of public key, last 20 bytes. Your on-chain account number. Share publicly — anyone can send funds to it.

0x742d35Cc6634C0532925a3b844Bc454e…

Seed phrase (BIP-39 mnemonic)

A human-readable backup of your private key — 12 or 24 words from the BIP-39 wordlist (2048 words). Deterministically regenerates all your keys across all compatible wallets. Whoever has these words controls all your funds across every chain. Write it on paper, store offline, never photograph it, never type it on any website.

grocery · window · planet · ocean · tiger · solar · dream · vault · crystal · bridge · ember · moon

Wallet types

Hot wallet

Software wallets

Connected to the internet. Convenient for daily use and dApp interactions. Higher attack surface — malicious sites can attempt to steal keys via browser extensions.

MetaMask · Phantom · Rabby · Coinbase Wallet · Trust Wallet

Cold wallet

Hardware wallets

Private key lives on an isolated security chip, never exposed to the internet. All transaction signing happens on-device. Best security for significant holdings.

Ledger · Trezor · Coldcard · Keystone

Layer 1 vs Layer 2 scaling

L1 — base layer

Settlement & security

The main chain (Ethereum, Bitcoin). Maximally decentralized and secure. Limited throughput causes high fees during peak demand. The ultimate source of truth.

L2 — scaling layer

Rollups execute off-chain

Execute thousands of transactions off-chain, post compressed proofs to L1 for security. 10–100× cheaper and faster. Two dominant types:
